# ТЗ — Sprint 3: Форма поиска

## 1. Контекст

API готово (Sprint 2). Sprint 3 строит пользовательский интерфейс: компонент `SearchForm` позволяет туристу выбрать аэропорт, курорт, дату и количество пассажиров, затем отправить запрос к `/api/calculate`. Результат (объект с ценой) передаётся родительскому компоненту для отображения в Sprint 4. Маршруты для дропдаунов загружаются из `GET /api/routes` при монтировании компонента.

## 2. Архитектура (ASCII)

```
[App.tsx]
  ↓ рендерит SearchForm
[SearchForm.tsx]
  ↓ GET /api/routes при монтировании → заполняет дропдауны
  ↓ POST /api/calculate при сабмите → передаёт результат в App
[App.tsx]
  ↓ передаёт результат в ResultsPage (Sprint 4)
```

## 3. Что входит в спринт

- Компонент `SearchForm` с 4 полями: аэропорт (select), курорт (select), дата (date input), пассажиры (number input)
- Дропдауны аэропортов и курортов заполняются из `GET /api/routes`
- Дропдаун курорта фильтруется по выбранному аэропорту
- Валидация: все поля обязательны, пассажиры 1–8
- При сабмите: `POST /api/calculate`, результат передаётся вверх через `onResult` prop
- Loading state во время запроса, error state при ошибке API

## 4. Out of scope

- Отображение результата (ResultsPage — Sprint 4)
- i18n / переводы
- Деплой (Sprint 4)
- Сохранение формы в localStorage
