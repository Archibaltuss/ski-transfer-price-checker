# План тестирования — Sprint 3

## Уровни тестирования

| Уровень | Инструмент | Что покрывает |
|---------|------------|---------------|
| Manual | Browser (оба сервера запущены) | UI взаимодействие, загрузка маршрутов, сабмит |
| Manual | Browser DevTools → Network | API-запросы, тела запросов и ответов |

## Тест-кейсы

### TC-3-01: Happy Path — Загрузка дропдаунов при открытии формы

**Предусловие:** оба сервера запущены, маршруты в routes.json

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть `http://localhost:5173` | `SearchForm` отображается |
| 2 | Проверить дропдаун «Аэропорт» | Содержит GVA и ZRH (или аналоги из routes.json) |
| 3 | Выбрать аэропорт GVA | Дропдаун «Курорт» показывает только курорты, доступные из GVA |

---

### TC-3-02: Happy Path — Успешный расчёт цены

**Предусловие:** оба сервера запущены

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Выбрать: Airport=GVA, Resort=Verbier, Date=2026-02-01, Passengers=2 | Кнопка «Calculate» активна |
| 2 | Нажать «Calculate» | Кнопка становится «Calculating...» и disabled |
| 3 | Дождаться ответа | `onResult` вызван, в Network виден POST /api/calculate с HTTP 200 |

---

### TC-3-03: Негативный — Сабмит с незаполненными полями

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть форму, не заполнять поля | Кнопка «Calculate» disabled |
| 2 | Попытаться кликнуть по disabled кнопке | Запрос к API не отправляется |

---

### TC-3-04: Негативный — passengers = 0

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Ввести 0 в поле Passengers | Inline-сообщение об ошибке: «Minimum 1 passenger» |
| 2 | Кнопка «Calculate» | Остаётся disabled |

---

### TC-3-05: Негативный — Ошибка API

**Предусловие:** бэкенд недоступен или возвращает 500

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Остановить бэкенд, заполнить и отправить форму | Под формой появляется сообщение об ошибке |
| 2 | Кнопка после ошибки | Возвращается в состояние «Calculate» (не locked) |

---

### TC-3-06: Негативный — Дата в прошлом

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Попытаться выбрать дату < сегодня в date input | Браузерная валидация блокирует выбор |

---

## Edge Cases

| # | Сценарий | Ожидаемое поведение |
|---|----------|---------------------|
| 1 | `/api/routes` недоступен при монтировании | Форма показывает сообщение об ошибке загрузки маршрутов |
| 2 | Смена аэропорта после выбора курорта | Дропдаун курорта сбрасывается |
| 3 | Двойной клик по «Calculate» | Второй запрос не отправляется (кнопка disabled) |

## Критерии приёмки тестирования

- [ ] TC-3-01 — TC-3-05 пройдены.
- [ ] Edge Cases верифицированы.
- [ ] E2E Happy Path: форма заполнена → API вызван → `onResult` получил данные.
