# План тестирования — Sprint 1

## Уровни тестирования

| Уровень | Инструмент | Что покрывает |
|---------|------------|---------------|
| Manual | Terminal | Запуск dev-сервера и сборка фронтенда |
| Manual | curl / браузер | Health-check эндпоинт бэкенда |
| Manual | Browser DevTools | CORS и связность фронтенда с бэкендом |

## Тест-кейсы

### TC-1-01: Happy Path — Запуск dev-сервера фронтенда

**Предусловие:** зависимости установлены (`npm install` в `frontend/`)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Выполнить `npm run dev` в `frontend/` | Сервер стартует на порту 5173, в консоли нет ошибок |
| 2 | Открыть `http://localhost:5173` в браузере | Страница отображается без ошибок в консоли |

---

### TC-1-02: Happy Path — Сборка фронтенда

**Предусловие:** зависимости установлены

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Выполнить `npm run build` в `frontend/` | Команда завершается с кодом 0 |
| 2 | Проверить наличие `frontend/dist/` | Папка существует и содержит `index.html` |

---

### TC-1-03: Happy Path — Health-check эндпоинт

**Предусловие:** бэкенд запущен (`uvicorn main:app --reload` в `backend/`)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Выполнить `curl http://localhost:8000/api/health` | HTTP 200, тело `{"status": "ok"}` |
| 2 | Открыть `http://localhost:8000/docs` в браузере | Swagger UI отображается, эндпоинт `/api/health` виден |

---

### TC-1-04: Happy Path — Вызов health-check с фронтенда без CORS-ошибок

**Предусловие:** бэкенд запущен (порт 8000) и фронтенд запущен (порт 5173)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть `http://localhost:5173`, открыть Browser DevTools → Console | Нет ошибок CORS |
| 2 | DevTools → Network → запрос к `/api/health` | Статус 200, ответ `{"status":"ok"}` |
| 3 | Проверить Console → лог из `App.tsx` | Сообщение с `{"status":"ok"}` |

---

### TC-1-05: Негативный — порт 8000 уже занят

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Попытаться запустить бэкенд, когда порт 8000 занят | uvicorn выводит ошибку `[Errno 10048]` / `address already in use` |

---

### TC-1-06: Негативный — CORS-блокировка для неразрешённого домена

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Выполнить `fetch` к `http://localhost:8000/api/health` напрямую с другого домена/порта | Браузер блокирует запрос, DevTools показывает CORS-ошибку |

---

## Edge Cases

| # | Сценарий | Ожидаемое поведение |
|---|----------|---------------------|
| 1 | `npm install` не выполнен перед `npm run dev` | node_modules отсутствует, понятная ошибка |
| 2 | `pip install` не выполнен перед `uvicorn` | Ошибка импорта fastapi с понятным traceback |
| 3 | Proxy не настроен, фронт обращается напрямую к `:8000` | CORS-ошибка в консоли браузера |

## Критерии приёмки тестирования

- [ ] Все TC (TC-1-01 — TC-1-04) пройдены.
- [ ] Edge Cases верифицированы.
