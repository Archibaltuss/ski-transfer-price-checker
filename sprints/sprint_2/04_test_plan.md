# План тестирования — Sprint 2

## Уровни тестирования

| Уровень | Инструмент | Что покрывает |
|---------|------------|---------------|
| Unit | pytest | Формула расчёта цены (calculate_price функция) |
| Integration | pytest + httpx / curl | /api/routes, /api/calculate с реальными JSON |
| Manual | curl / Swagger UI | Негативные сценарии, граничные значения |

## Тест-кейсы

### TC-2-01: Happy Path — GET /api/routes возвращает все маршруты

**Предусловие:** бэкенд запущен, `routes.json` содержит минимум 6 маршрутов

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | `GET /api/routes` | HTTP 200 |
| 2 | Проверить тело ответа | JSON-массив, минимум 6 элементов, каждый содержит `id`, `from_airport`, `to_resort` |
| 3 | Проверить наличие GVA и ZRH маршрутов | Оба аэропорта присутствуют |

---

### TC-2-02: Happy Path — POST /api/calculate, 1 пассажир

**Предусловие:** бэкенд запущен

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | `POST /api/calculate` с `{"from_airport": "GVA", "to_resort": "VERBIER", "passengers": 1, "date": "2026-01-15"}` | HTTP 200 |
| 2 | Проверить `price_chf` | Число > 0, ≤ 2 десятичных знака |
| 3 | Проверить поля ответа | `route_name`, `distance_km`, `passengers: 1`, `currency: "CHF"` присутствуют |

---

### TC-2-03: Happy Path — POST /api/calculate, групповой тариф (4 пассажира)

## Edge Cases

| # | Сценарий | Ожидаемое поведение |
|---|----------|---------------------|
| 1 | `passengers = 8` (максимум) | Применяется наименьший коэффициент, HTTP 200 |
| 2 | `from_airport` в нижнем регистре (`"gva"`) | 404 или case-insensitive поиск (задокументировать поведение) |
| 3 | `date` в прошлом | Цена считается нормально (дата не влияет на цену в MVP) |
| 4 | Недостающие поля в теле запроса | HTTP 422 с описанием валидационной ошибки |
