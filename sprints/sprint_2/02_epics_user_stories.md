# Эпики и User Stories — Sprint 2

---

## EPIC 2.1: Данные маршрутов и ценообразования

Цель: создать JSON-файлы с маршрутами и тарифной сеткой как единственный источник данных приложения.

---

### US 2.1.1 — Создать routes.json с маршрутами аэропорт → курорт

**Как** разработчик,
**я хочу** чтобы данные о маршрутах хранились в `backend/data/routes.json`,
**чтобы** бэкенд мог вычислять расстояние и загружать их в память при старте.

**Агент:** `[backend]`

Зависит от: нет (данные не зависят от кода)

Acceptance criteria:
- [ ] Файл `backend/data/routes.json` создан
- [ ] Содержит минимум 6 маршрутов, каждый с полями: `id`, `from_airport` (код IATA), `from_name`, `to_resort`, `to_name`, `distance_km`
- [ ] Охвачены минимум 2 аэропорта (GVA, ZRH) и 3 курорта (Verbier, Zermatt, Davos)
- [ ] JSON валиден (парсится без ошибок)

---

### US 2.1.2 — Создать pricing.json с тарифной сеткой

**Как** разработчик,
**я хочу** чтобы базовая ставка и групповые коэффициенты хранились в `backend/data/pricing.json`,
**чтобы** формулу можно было менять без правки кода.

**Агент:** `[backend]`

Зависит от: нет

Acceptance criteria:
- [ ] Файл `backend/data/pricing.json` создан
- [ ] Содержит `base_rate_chf_per_km` (число) и `group_multipliers` (объект: ключ = макс. число пассажиров, значение = коэффициент)
- [ ] Покрывает диапазон 1–8 пассажиров
- [ ] JSON валиден

---

## EPIC 2.2: API для расчёта и справочник маршрутов

Цель: предоставить фронтенду два эндпоинта: список маршрутов для дропдаунов и расчёт стоимости.

---

### US 2.2.1 — Реализовать GET /api/routes

**Как** фронтенд-разработчик (Sprint 3),
**я хочу** получать список всех маршрутов через `GET /api/routes`,
**чтобы** заполнить дропдауны аэропортов и курортов в SearchForm.

**Агент:** `[backend]`

Зависит от: US 2.1.1

Acceptance criteria:
- [ ] `GET /api/routes` возвращает HTTP 200
- [ ] Тело ответа — JSON-массив объектов с полями: `id`, `from_airport`, `from_name`, `to_resort`, `to_name`
- [ ] Данные загружаются из `routes.json` при старте, не читаются с диска при каждом запросе

---

### US 2.2.2 — Реализовать POST /api/calculate

**Как** турист,
**я хочу** получить стоимость трансфера, отправив аэропорт, курорт и количество пассажиров,
**чтобы** мгновенно оценить цену без обращения к реальному оператору.

**Агент:** `[backend]`

Зависит от: US 2.1.1, US 2.1.2

Acceptance criteria:
- [ ] `POST /api/calculate` принимает JSON-тело: `{ "from_airport": str, "to_resort": str, "passengers": int, "date": str }`
- [ ] Возвращает HTTP 200 с полями: `price_chf` (float, округлён до 2 знаков), `route_name` (str), `distance_km` (int), `passengers` (int), `currency` (="CHF")
- [ ] Формула: `price = distance_km × base_rate × group_multiplier(passengers)`
- [ ] Если маршрут не найден — возвращает HTTP 404 с `{"detail": "Route not found"}`
- [ ] Если `passengers` < 1 или > 8 — возвращает HTTP 422
