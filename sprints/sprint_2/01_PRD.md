# ТЗ — Sprint 2: Данные и логика расчёта

## 1. Контекст

Инфраструктура готова (Sprint 1). Sprint 2 добавляет смысловую начинку: данные о маршрутах и ценообразовании хранятся в JSON-файлах внутри бэкенда. Бэкенд вычисляет стоимость трансфера по формуле: расстояние × базовая ставка × групповой коэффициент. Фронтенд получает данные через REST API — нет прямого доступа к файлам.

## 2. Архитектура (ASCII)

```
[Frontend: React :5173]
       ↓ POST /api/calculate (JSON)
       ↓ GET  /api/routes
[Backend: FastAPI :8000]
       ↓ читает при старте
[backend/data/routes.json]   — маршруты аэропорт → курорт
[backend/data/pricing.json]  — базовая ставка + групповые коэффициенты
```

## 3. Что входит в спринт

- Создать `backend/data/routes.json` — минимум 6 маршрутов (2 аэропорта × 3 курорта)
- Создать `backend/data/pricing.json` — базовая ставка (CHF/км) + коэффициенты по числу пассажиров
- Реализовать `GET /api/routes` → список маршрутов (для дропдаунов Sprint 3)
- Реализовать `POST /api/calculate` → стоимость трансфера
- Данные загружаются из JSON-файлов при старте приложения, не читаются при каждом запросе

## 4. Out of scope

- UI форма поиска (Sprint 3)
- Страница результатов и деплой (Sprint 4)
- Реальные цены перевозчиков, динамическое ценообразование
- Авторизация, БД, логирование транзакций
